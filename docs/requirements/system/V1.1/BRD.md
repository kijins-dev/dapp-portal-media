# BRD: Dapp Portal Media V1.1（要求定義書）

> V1.0からの変更: Codex/Geminiレビュー指摘を反映。段階リリース導入、KPI漏斗精緻化、コンプライアンス要件追加。

## 1. 背景・目的

LINE Dapp Portal（Mini Dapp）は2025年1月にローンチされたWeb3ゲームプラットフォームで、2026年3月時点で77本のアプリが公開されている。2025年末までに1,000本以上のリリースが予定されており、急速に拡大中。

しかし、個別ゲームの日本語攻略記事はBombie/Cattea/キャプテン翼の3本程度しか存在せず、残り70本以上は記事がゼロという状態（2026年3月調査）。

この空白市場に対して、AIによる記事自動生成 + 自動公開の仕組みを構築し、ゲーム攻略メディアとして先行ポジションを確立する。

## 2. ステークホルダー

| 役割 | 誰 | 関心事 |
|------|------|--------|
| オーナー/運営者 | 自分 | 月30万円の収益化、運用工数の最小化 |
| 読者（ユーザー） | Dapp Portalのプレイヤー（日本語圏） | ゲームの遊び方・攻略情報・稼ぎ方を知りたい |
| 広告主/ASP | TCSアフィリエイト、アクセストレード等 | 仮想通貨取引所への送客 |
| ゲーム開発元 | Mini Dapp開発者 | 自社ゲームの認知拡大 |

## 3. ユーザーストーリー

### 読者（プレイヤー）
- Dapp Portalの新しいゲームを始めたプレイヤーとして、遊び方と攻略のコツを知りたい。なぜなら公式の説明だけではわからないことが多いから。
- ゲームで稼いだKAIAを日本円に換金したいプレイヤーとして、換金方法と必要な準備を知りたい。なぜなら仮想通貨の取引が初めてで手順がわからないから。
- 新しいゲームを探しているプレイヤーとして、おすすめのMini Dappランキングや比較情報が欲しい。なぜなら77本以上ある中から自分に合うものを選べないから。

### 運営者（自分）
- メディア運営者として、新規ゲーム追加時に自動で記事が生成・公開されてほしい。なぜなら手動で1,000本の記事を書くのは不可能だから。
- メディア運営者として、記事経由で仮想通貨取引所の口座開設に繋げたい。なぜなら月30万円の収益目標を達成する主要な収益源だから。

## 4. ビジネスゴール・KPI

### 収益目標
- **月間30万円**（安定運用時・12ヶ月目目標）

### 収益構造
| 収益源 | 単価 | 月額目標 |
|--------|------|---------|
| 仮想通貨取引所アフィリエイト（メイン） | 3,000〜7,000円/件 | 30万円 |
| ゲーム紹介リファラル報酬（サブ） | 数円〜数十円/件 | +α |

### KPI漏斗（精緻化）

```
月間PV（攻略記事）: 30,000
  × 換金ガイドへの遷移率: 15%
  = 換金ガイドPV: 4,500
    × アフィリンククリック率: 5%
    = クリック数: 225
      × 口座開設完了率: 20%
      = 成約件数: 45件
        × 平均単価: 7,000円
        = 月間収益: 315,000円
```

### KPIフェーズ別推移

| 指標 | Phase 0（〜3ヶ月） | Phase 1（4〜6ヶ月） | Phase 2（7〜12ヶ月） |
|------|-------------------|-------------------|---------------------|
| 公開記事数 | 50本 | 200本 | 500本+ |
| 月間PV | 1,000 | 8,000 | 30,000 |
| アフィ成約数/月 | 0件（ASP未申請） | 10件 | 45件 |
| 月間収益 | 0円 | 7万円 | 30万円 |
| 主な活動 | サイト構築、記事蓄積 | ASP申請、導線最適化 | 自動化完成、拡大 |

※SEOが軌道に乗るまで3〜6ヶ月かかる前提で設計。Phase 0は収益ゼロを許容。

### 計測設計
- Google Search Console: 検索流入、表示回数、CTR、平均順位を記事単位で計測
- GA4: PV、ページ遷移率（攻略→換金ガイド）、イベントトラッキング（アフィリンククリック）
- ASP管理画面: 成約数、承認率を月次で確認

## 5. 制約条件

### 技術的制約
- unifi.meはSPA。Playwright + UA偽装でスクレイピング可能（実証済み）
- スクレイピング不可時の代替: 公式API確認 → 公開情報のみ利用 → 手動データ収集
- 記事生成にはClaude API使用（API費用発生、モデル選定でコスト最適化）
- 静的サイトホスティングは無料枠で運用可能（Cloudflare Pages）

### 法的・規約上の制約
- **スクレイピング**: 実装前にunifi.meのrobots.txt・利用規約を精査。禁止されている場合は公開情報のみ利用に切り替え
- ASP（TCSアフィリエイト等）の審査あり。Phase 0でサイトを育ててからPhase 1で申請
- 景品表示法・金融商品取引法に抵触しない表現必須

### コンプライアンス要件（V1.1追加）
- 禁止表現辞書の整備（「必ず儲かる」「元本保証」等のNGワード）
- 免責テンプレートの設置（「投資は自己責任」等）
- 換金ガイド・口座開設関連の記事は**手動レビュー必須**（自動公開対象外）
- 金融商品に関する断定的表現のフィルタリング

### 運用上の制約
- 手動作業は最小限（目標）だが、月5時間程度の保守作業を見込む
- 保守内容: スクレイピング不具合対応、記事品質チェック、ASP管理

## 6. スコープ

### 段階リリース計画（V1.1変更）

#### Phase 0: 基盤構築（1〜3ヶ月目）
- [x] unifi.me/apps のスクレイピング
- [ ] 新規ゲーム検知（差分比較）
- [ ] 記事自動生成（Claude API）→ **下書きとして出力**
- [ ] 手動レビュー後にサイト公開
- [ ] Astroサイト構築 + Cloudflare Pagesデプロイ
- [ ] サイトマップ自動生成
- [ ] Search Console + GA4 導入

#### Phase 1: 半自動運用（4〜6ヶ月目）
- [ ] 攻略記事の自動公開（コンプライアンスチェック通過分のみ）
- [ ] 換金ガイド記事は手動レビュー維持
- [ ] ASPへの申請・アフィリエイトリンク設置
- [ ] リファラルリンク埋め込み
- [ ] GA4イベントトラッキング（クリック計測）

#### Phase 2: 全自動運用（7〜12ヶ月目）
- [ ] 高信頼ソースの記事は全自動公開
- [ ] 週次ランキング・まとめ記事の自動生成
- [ ] 導線最適化（CTR/CVR改善）
- [ ] 記事更新の自動化（プレイ数等のデータ更新）

### やらないこと（全Phase共通）
- 英語版・多言語対応
- 動画コンテンツの自動生成
- SNS自動投稿（X, LINE等）
- 有料コンテンツ販売
- ゲームプレイの自動スクショ取得
- ゲーム自体の自動プレイ/Bot
