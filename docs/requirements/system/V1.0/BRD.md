# BRD: Dapp Portal Media（要求定義書）

## 1. 背景・目的

LINE Dapp Portal（Mini Dapp）は2025年1月にローンチされたWeb3ゲームプラットフォームで、2026年3月時点で77本のアプリが公開されている。2025年末までに1,000本以上のリリースが予定されており、急速に拡大中。

しかし、個別ゲームの日本語攻略記事はBombie/Cattea/キャプテン翼の3本程度しか存在せず、残り70本以上は記事がゼロという状態（2026年3月調査）。

この空白市場に対して、AI による記事自動生成 + 自動公開の仕組みを構築し、ゲーム攻略メディアとして先行ポジションを確立する。

## 2. ステークホルダー

| 役割 | 誰 | 関心事 |
|------|------|--------|
| オーナー/運営者 | 自分 | 月30万円の収益化、運用工数の最小化 |
| 読者（ユーザー） | Dapp Portalのプレイヤー（日本語圏） | ゲームの遊び方・攻略情報・稼ぎ方を知りたい |
| 広告主/ASP | TCSアフィリエイト、アクセストレード等 | 仮想通貨取引所への送客 |
| ゲーム開発元 | Mini Dapp開発者 | 自社ゲームの認知拡大 |

## 3. ユーザーストーリー

### 読者（プレイヤー）
- Dapp Portalの新しいゲームを始めたプレイヤーとして、遊び方と攻略のコツを知りたい。なぜなら公式の説明だけではわからないことが多いから。
- ゲームで稼いだKAIAを日本円に換金したいプレイヤーとして、換金方法と必要な準備を知りたい。なぜなら仮想通貨の取引が初めてで手順がわからないから。
- 新しいゲームを探しているプレイヤーとして、おすすめのMini Dappランキングや比較情報が欲しい。なぜなら77本以上ある中から自分に合うものを選べないから。

### 運営者（自分）
- メディア運営者として、新規ゲーム追加時に自動で記事が生成・公開されてほしい。なぜなら手動で1,000本の記事を書くのは不可能だから。
- メディア運営者として、記事経由で仮想通貨取引所の口座開設に繋げたい。なぜなら月30万円の収益目標を達成する主要な収益源だから。

## 4. ビジネスゴール・KPI

### 収益目標
- **月間30万円**（安定運用時）

### 収益構造

| 収益源 | 単価 | 必要件数/月 | 月額 |
|--------|------|------------|------|
| 仮想通貨取引所アフィリエイト（メイン） | 3,000〜7,000円/件 | 43〜100件 | 30万円 |
| ゲーム紹介リファラル報酬（サブ） | 数円〜数十円/件 | - | +α |

### KPI

| 指標 | 3ヶ月目 | 6ヶ月目 | 12ヶ月目 |
|------|---------|---------|----------|
| 公開記事数 | 100本 | 300本 | 700本+ |
| 月間PV | 3,000 | 10,000 | 20,000+ |
| アフィ成約数/月 | 5件 | 20件 | 43件+ |
| 月間収益 | 3.5万円 | 14万円 | 30万円+ |

### 収益達成ロジック
```
読者の導線:
ゲーム攻略記事（SEO流入）
  → 記事内「稼いだKAIAを換金するには取引所口座が必要」
  → 口座開設リンク（アフィリエイト成果発生）

逆算:
月30万 ÷ 7,000円/件 = 43件/月の口座開設
43件 ÷ CVR 0.5% = 8,600 PV/月
700記事 × 平均30 PV/記事 = 21,000 PV/月 → 十分到達可能
```

## 5. 制約条件

### 技術的制約
- unifi.me（Dapp Portal）はSPA。Playwright + UA偽装でスクレイピング可能（実証済み）
- 記事生成にはClaude API を使用（API費用が発生）
- 静的サイトホスティングは無料枠で運用可能（Cloudflare Pages）

### 法的・規約上の制約
- ASP（TCSアフィリエイト等）の審査あり。サイトがある程度育ってから申請
- 景品表示法・金融商品取引法に抵触しない表現（「必ず儲かる」等はNG）
- unifi.meのスクレイピングは利用規約を確認する必要あり

### 品質上の制約（Google SEO）
- サイトの主語は「ゲーム攻略メディア」（エンタメカテゴリ）とする
- YMYL（仮想通貨投資）に該当するコンテンツはサイト全体の一部に留める
- AI生成記事に独自データ（ユーザー数推移、リワード額等）を必ず含め差別化する
- 最低1,500文字/記事
- 競合ゼロの今はスピード優先、競合出現後に品質強化

### 運用上の制約
- 運営者の手動作業は最小限（目標: ゼロに近づける）
- 記事の質が低すぎる場合のリカバリー手段を用意する
