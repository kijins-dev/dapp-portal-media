# レビュー統合結果（V1.0）

## レビュー実施
- **Codex (GPT-5.3)**: Conditional Go
- **Gemini CLI**: Conditional Go

## 致命的な問題（3件）

### 1. Google Indexing APIの前提が不正確（Codex）
Google Indexing APIは`JobPosting`と`BroadcastEvent`ページ向け。通常のブログ記事には使えない。

**対応**: Indexing APIを削除。代替としてsitemap.xml自動更新 + Search Console ping + 通常クロール待ちに変更。

### 2. スクレイピングの法的・利用規約リスク（Codex/Gemini共通）
UA偽装+Bot検知回避を要件化しており、対象サイト規約次第で即停止リスク。代替手段が未定義。

**対応**: robots.txt確認を必須化。利用規約精査を実装前タスクに追加。スクレイピング不可時の代替手段（公式API確認、手動データ収集、公開情報のみ利用）を定義。

### 3. YMYL回避方針と収益導線の衝突（Codex）
エンタメ寄りを掲げつつ、換金・口座開設導線が中核。金融情報の誤情報リスク。

**対応**: コンプライアンスゲートを追加。禁止表現辞書、免責テンプレート、金融表現のNGワードフィルタを要件化。換金ガイドは手動レビュー必須に。

## 重要な指摘（6件）

### 4. KPI漏斗が粗い（Codex/Gemini共通）
PV→CV直結で算定。攻略記事→換金ガイド遷移→広告クリック→成約の各段階離脱が未考慮。

**対応**: 漏斗を精緻化。中間KPI（ガイド遷移率、クリック率）を追加。

### 5. V1.0スコープ過大（Codex）
全機能同時実装は高リスク。

**対応**: V0（スクレイピング+下書き生成）→ V1（半自動公開）→ V1.5（自動公開）に段階化。

### 6. APIコスト見積もり不足（Gemini）
記事数増加時のClaude API費用が予算超過の可能性。

**対応**: 1記事あたりのトークン数 × 単価でコストシミュレーション追加。モデル選定（Haiku/Sonnet）の方針明記。

### 7. 完全自動の現実性（Codex/Gemini共通）
外部サイトUI変更、API仕様変更等で停止リスク。

**対応**: 「目標はゼロだが月5時間の保守を見込む」を明記。監視・通知・復旧フロー定義。

### 8. データ設計の監査性不足（Codex）
source_url, fetched_at, content_hash等がない。

**対応**: スクレイピングデータに監査フィールド追加。

### 9. Cloudflare Pages無料プランでSLA不整合（Codex）
無料プランにSLA保証はない。

**対応**: SLA記述を削除。「無料プランのため、SLA保証なし。静的サイトの特性上、高可用性が期待できる」に修正。

## 改善提案（採用分）
- 計測設計先行（Search Console + GA4 + ASP成約の紐付け）
- UX導線強化（常設バナー、フローティングCTA）
- SEO成長曲線の現実的見直し（3-6ヶ月かかる前提）
