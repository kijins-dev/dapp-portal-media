# レビュープロンプト: Dapp Portal Media

以下のBRD（要求定義書）とSRS（要件定義書）をレビューしてください。

---

## BRD（要求定義書）

### 1. 背景・目的
LINE Dapp Portal（Mini Dapp）は2025年1月にローンチされたWeb3ゲームプラットフォームで、2026年3月時点で77本のアプリが公開されている。2025年末までに1,000本以上のリリースが予定されており、急速に拡大中。

しかし、個別ゲームの日本語攻略記事はBombie/Cattea/キャプテン翼の3本程度しか存在せず、残り70本以上は記事がゼロという状態（2026年3月調査）。

この空白市場に対して、AIによる記事自動生成 + 自動公開の仕組みを構築し、ゲーム攻略メディアとして先行ポジションを確立する。

### 2. ステークホルダー
- オーナー/運営者（自分）: 月30万円の収益化、運用工数の最小化
- 読者（ユーザー）: Dapp Portalのプレイヤー（日本語圏）。ゲームの遊び方・攻略情報・稼ぎ方を知りたい
- 広告主/ASP: TCSアフィリエイト、アクセストレード等。仮想通貨取引所への送客
- ゲーム開発元: Mini Dapp開発者。自社ゲームの認知拡大

### 3. ユーザーストーリー
- Dapp Portalの新しいゲームを始めたプレイヤーとして、遊び方と攻略のコツを知りたい。なぜなら公式の説明だけではわからないことが多いから。
- ゲームで稼いだKAIAを日本円に換金したいプレイヤーとして、換金方法と必要な準備を知りたい。なぜなら仮想通貨の取引が初めてで手順がわからないから。
- 新しいゲームを探しているプレイヤーとして、おすすめのMini Dappランキングや比較情報が欲しい。なぜなら77本以上ある中から自分に合うものを選べないから。
- メディア運営者として、新規ゲーム追加時に自動で記事が生成・公開されてほしい。なぜなら手動で1,000本の記事を書くのは不可能だから。

### 4. ビジネスゴール・KPI
- 収益目標: 月間30万円（安定運用時）
- メイン収益: 仮想通貨取引所アフィリエイト（単価3,000〜7,000円/件）
- サブ収益: ゲーム紹介リファラル報酬（+α）
- CVR: 0.5%想定
- 必要PV: 月間8,600〜20,000PV
- KPI推移: 3ヶ月目100記事/3,000PV → 6ヶ月目300記事/10,000PV → 12ヶ月目700記事/20,000PV
- 読者の導線: ゲーム攻略記事（SEO流入）→「KAIAを換金するには取引所口座が必要」→ 口座開設リンク（アフィリエイト成果）

### 5. 制約条件
- 技術: unifi.meはSPA（Playwright+UA偽装でスクレイピング実証済み）、Claude API費用発生、ホスティングは無料枠
- 法的: ASP審査あり（サイト育成後）、景表法・金商法準拠の表現、スクレイピング規約確認要
- SEO品質: サイトの主語は「ゲーム攻略メディア」（エンタメ）。YMYL直撃を回避。AI生成記事に独自データ（ユーザー数、リワード額）を含め差別化。最低1,500文字/記事。競合ゼロの今はスピード優先。
- 運用: 手動作業は最小限（目標ゼロ）

### 6. スコープ
**やること（V1.0）**: スクレイピング、新規ゲーム検知、記事自動生成（Claude API）、静的サイト構築（Astro）、自動デプロイ（GitHub Actions + Cloudflare Pages）、アフィリエイトリンク埋め込み、リファラルリンク埋め込み、サイトマップ + Google Indexing API

**やらないこと**: 多言語対応、動画生成、SNS自動投稿、有料コンテンツ販売、ゲームプレイ自動スクショ、ゲーム自動プレイBot

## SRS（要件定義書）

### 1. 機能要件

#### スクレイパー
- Playwright（Node.js）でunifi.me/appsをスクレイピング。UA偽装+SPA待機でBot検知回避
- 取得データ: アプリ名、カテゴリ、プレイ数、サムネイルURL
- 人気順・新着順の両方を取得
- 前回データとの差分比較で新規ゲーム検知
- GitHub Actions cron（1日1回）で実行
- 個別ゲーム詳細ページからリワードミッション情報も取得
- Web検索で各ゲームの補完情報を収集

#### 記事生成エンジン
- スクレイピングデータ + Web検索情報 + テンプレート + プロンプト → Claude API → Markdown出力
- 記事テンプレート構成: ゲーム概要 → 始め方 → 遊び方 → 攻略のコツ → 報酬・稼ぎ方 → KAIA換金方法 → まとめ
- 品質基準: 最低1,500文字、独自データ必須、テンプレート5パターン以上、NGワードフィルタ
- 記事種類: 個別攻略（随時）、ランキング（週1）、換金ガイド（月1）、新着まとめ（週1）

#### サイト（Astro）
- ページ: トップ、個別攻略（/games/{slug}/）、カテゴリ一覧、ランキング、換金ガイド、Dapp Portal解説
- SEO: メタタグ自動生成、構造化データ（JSON-LD）、sitemap.xml、OGP画像、Google Indexing API、内部リンク
- アフィリエイト: 換金ガイドに取引所口座開設リンク、各記事から換金ガイドへ内部リンク、設定ファイルで一元管理
- リファラル: 各記事の「始め方」セクションに紹介リンク埋め込み

#### CI/CDパイプライン
- 毎日: スクレイピング → 新規検知 → 情報収集 → 記事生成 → コミット → 自動デプロイ → インデックス通知
- 週次: ランキング更新、新着まとめ生成
- エラー時: リトライ3回、Slack通知、失敗リスト記録

### 2. 非機能要件
- 性能: スクレイピング5分以内、記事生成30秒/本、Lighthouse 90以上
- 可用性: Cloudflare Pages SLA 99.9%
- セキュリティ: APIキーはGitHub Secrets、アフィリンクは設定ファイル管理
- 保守性: テンプレート・セレクタ・メタデータすべて設定ファイルで管理
- コスト: 月額約3,000円（Claude API + ドメイン）

### 3. データ設計
- スクレイピングデータ: JSON形式（app名、slug、カテゴリ、プレイ数、サムネURL、初回検知日、リワード情報）
- 記事メタデータ: Markdown frontmatter（title、slug、category、play_count、published_at、referral_link、tags、description）
- 設定ファイル: YAML（アフィリエイトリンク、スクレイパー設定）

### 4. 技術スタック
TypeScript、Playwright、Claude API（Anthropic SDK）、Astro、Cloudflare Pages、GitHub Actions、Google Indexing API

### 5. 画面仕様
- トップ: 最新記事カード + 人気ランキング + カテゴリナビ
- 記事: パンくず + 目次 + 本文 + サイドバー（関連ゲーム・換金導線）+ 記事下CTA
- モバイルファースト、ダーク/ライトモード

### 6. 外部連携
unifi.me（スクレイピング）、Claude API（記事生成）、Google Indexing API（インデックス）、Cloudflare Pages（ホスティング）、TCSアフィリエイト（収益化）

## レビュー観点

### 要求定義（BRD）の観点
1. **目的の明確性**: プロジェクトの目的とビジネス価値は明確か
2. **ユーザーストーリーの網羅性**: 主要なユーザーの利用シナリオを網羅しているか
3. **スコープの妥当性**: V1.0のスコープは大きすぎないか、小さすぎないか
4. **ビジネスゴールの測定可能性**: KPIは現実的で測定可能か。月30万円の収益計算ロジックに穴はないか

### 要件定義（SRS）の観点
1. **要求との整合性**: BRDの要求をSRSが漏れなくカバーしているか
2. **データ設計**: スクレイピングデータ構造、記事メタデータの設計は適切か
3. **技術適合性**: Astro + Cloudflare Pages + GitHub Actionsの組み合わせに問題はないか
4. **UX**: 読者がゲーム攻略 → 換金 → 口座開設に自然に流れる導線になっているか
5. **運用面**: 全自動パイプラインの実現可能性。GitHub Actionsの無料枠で足りるか。障害時のリカバリー
6. **セキュリティ**: APIキー管理、スクレイピングの法的リスク
7. **矛盾・不整合**: BRDとSRSの間、またはSRS内部に矛盾がないか

### 出力形式
以下の形式で回答してください:
1. **致命的な問題**（あれば）: 進行を止めるべき問題
2. **重要な指摘**: 修正すべき問題
3. **改善提案**: あるとより良くなる提案
4. **良い点**: 評価できるポイント
5. **総合評価**: Go / Conditional Go / No Go
