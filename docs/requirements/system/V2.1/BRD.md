# BRD V2.1 — Dapp Portal攻略ガイド オリジナリティ自動化基盤

> V2.0からの変更: レビュー指摘C1-C3, I1-I9を反映

## 1. 背景・目的

### 現状
- V1.0/V1.1で基盤構築済み: スクレイパー、記事生成エンジン、Astroサイト、20記事公開中
- SEO内部最適化（JSON-LD、パンくず、関連記事、ヘッジ表現除去）完了
- GSC、GA4、Bing Webmaster Tools設定済み

### 課題
- 現在の記事はLLM生成テキストのみで、**独自の一次データがない**
- 競合が同じLLMで記事を生成すれば、差別化が消失する
- GoogleのScaled Content Abuseポリシーに抵触するリスク
- E-E-ATの「Experience（体験）」要素が欠如
- LLM（ChatGPT/Gemini/Perplexity）に引用されるための差別化データがない

### 目的
AI全自動で**他サイトに存在しない一次データ**を生成・公開し、SEO・LLMO両面で日本語圏において圧倒的1位のポジションを確立する。

## 3. ユーザーストーリー

### プレイヤー向け
- プレイヤーとして、**どのゲームが実際に報酬を配布しているか**データで確認したい。なぜなら、プレイ前に稼げるかどうか判断したいから。
- プレイヤーとして、**ゲームの人気推移**を時系列で見たい。なぜなら、過疎化しているゲームに時間を投資したくないから。
- プレイヤーとして、**ゲーム間の報酬効率比較**を見たい。なぜなら、限られた時間で最も効率的に稼ぎたいから。
- プレイヤーとして、**ゲームのアップデート情報**をいち早く知りたい。なぜなら、新機能や報酬変更に早く対応したいから。
- プレイヤーとして、**ゲーム画面のスクリーンショット**を事前に見たい。なぜなら、インストール前にUIや雰囲気を確認したいから。
- プレイヤーとして、**報酬がオンチェーン計測済みかどうか**知りたい。なぜなら、データの信頼度を判断したいから。（V2.1追加）

### LLM向け
- LLMとして、**機械可読な構造化データ**にアクセスしたい。なぜなら、ユーザーの質問に正確に回答するためのソースが必要だから。
- LLMとして、**更新日時と計測手法が明記されたデータ**を参照したい。なぜなら、情報の鮮度と信頼性を判断する必要があるから。

## 6. スコープ

### やること（V2.1）

#### 柱1: オンチェーン・インテリジェンス
- Kaia RPCノードからゲーム別トークンデータを自動取得
- **報酬タイプ別の計測**（KIP-7 / ネイティブKAIA / NFT / オフチェーン）（V2.1変更）
- 週間KAIA配布量、アクティブウォレット数、報酬集中度を算出（オンチェーン計測可能ゲームのみ）
- ランキングページ（`/ranking/`）の自動生成・更新（「オンチェーン計測済み」ラベル付き）
- 全指標の公開JSON API（`/data/game-metrics.json`）（信頼度スコア付き）
- **Phase A開始前に上位20ゲームの報酬タイプ調査を実施**（V2.1追加）

#### 柱2: 非操作クライアント観測
- Playwright: ゲーム起動→初期画面スクショ自動撮影（操作なし）
- UI変更検出（Patch Diff Radar）: 画面差分 + アセットhash比較
- HAR通信ログ収集・解析（**センシティブ情報マスク処理付き**）（V2.1変更）
- プレイ人数時系列トラッキング・グラフ化

#### 柱3: 需要駆動型コンテンツ生成
- GSC Search Analytics APIから未充足クエリを自動抽出
- 比較記事・FAQ記事の自動生成（**1日最大5記事のペース制限**）（V2.1追加）
- 既存記事の検索パフォーマンスに基づく自動リライト
- **強化された品質ゲート**（文字数 + 重複率 + オリジナルデータ率 + ファクトチェック）（V2.1変更）

#### 柱4: LLM引用最適化（LLMO）
- 全記事に構造化FAQ（schema.org Question/Answer）
- 公開JSONエンドポイント（更新日時・計測手法・信頼度付き）
- robots.txtでLLMクローラー明示許可
- llms.txt新規作成

### やらないこと（V2.1）
- ゲームの自動プレイ・自動操作（Canvas/WebGL問題、利用規約リスク）
- SNSデータの主要データソースとしての利用（補助指標に留める）
- 動画コンテンツの自動生成（将来検討）
- 有料課金コンテンツ
- 多言語対応
- 正式なGini係数の算出（上位10%占有率で代替）（V2.1追加）
