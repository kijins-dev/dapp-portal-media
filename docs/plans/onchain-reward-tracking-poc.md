## 1. RPCクエリの動作確認

### 使用エンドポイント
- **Kaia Public RPC**: `https://public-en.node.kaia.io`
- **プロトコル**: JSON-RPC 2.0 (EVM互換)
- **認証**: 不要（公開ノード）

### レスポンス時間

| クエリ種別 | 結果件数 | レスポンス時間 |
|-----------|---------|-------------|
| `eth_blockNumber` | 1件 | 116-122ms |
| `eth_getLogs` 1週間 (BOMB) | 2件 | 231-346ms |
| `eth_getLogs` 1週間 (ELDE) | 221件 | 274-898ms |
| `eth_getLogs` 1週間 (SLX) | 87件 | 441ms |
| `eth_getLogs` 2ヶ月 (BOMB) | 2,121件 | 1,162-3,645ms |
| `eth_call` (トークン情報) | 1件 | ~120ms |

### 制限事項
- `eth_getLogs` の結果が **10,000件を超えるとエラー** (`query returned more than 10000 results`)
- fromBlock=0x0 からの全期間クエリは **タイムアウト**（10秒超）
- ブロック範囲を分割してチャンク取得する必要あり
- Kaia は約1秒/ブロック → 1週間 = ~604,800ブロック

### 紐付け方法の実現性

**課題:**
- ゲーム名とトークン名が一致しないケース（Bombie→BOMB, Captain Tsubasa→JOHN）
- CoinGecko無料APIのレート制限（30リクエスト/分）
- トークン未発行のゲームが大半（53ゲーム中47は未紐付け）

## 5. 週次データ収集の実現性評価

### 推定処理時間（週次バッチ）

| 処理 | 1トークンあたり | 6トークン合計 |
|-----|-------------|------------|
| eth_call (基本情報) | ~500ms | ~3秒 |
| eth_getLogs (1週間) | ~500ms | ~3秒 |
| データ解析 | ~100ms | ~600ms |
| **合計** | ~1.1秒 | **~7秒** |

→ 週次バッチは **全く問題ない**。1分以内に完了。

### 10,000件制限への対処

アクティブなトークン（SLX, ELDE）でも1週間で数百件程度なので、**1週間チャンクなら制限に引っかからない**。
ただし、将来トークンが増えた場合やエアドロップイベント時は、1日チャンクに切り替える必要あり。

### 推奨アーキテクチャ

```
[Weekly Cron Job]
    |
    v
[Token Registry] → CoinGecko APIで定期更新
    |
    v
[RPC eth_getLogs] → 1週間分のTransferイベント取得
    |
    v
[Decode & Analyze]
    - from/to/amount デコード
    - 報酬配布パターン検出（1-to-many送金）
    - Mint/Burn集計
    |
    v
[Store] → JSON or DB
```

## 6. 技術的ブロッカー

### 現時点で判明しているもの

1. **KaiaScan OAPI は有料/認証必須**
   - 公開APIは存在するが、`UNAUTHORIZED_ACCESS` エラーが返る
   - 無料プラン登録で基本的なトークン情報は取得可能とみられる
   - ただし**RPCで代替可能**なので、ブロッカーではない

2. **報酬配布アドレスの特定が困難**
   - EOAからの直接送金はTransferイベントに含まれる
   - しかし「どのアドレスがゲーム開発者のもの」かの公式情報がない
   - ヒューリスティック（1-to-many、高頻度、MintのTO先）で推定するしかない

3. **トークン未発行ゲームが大半**
   - apps.json 53ゲーム中、Kaiaトークンが確認できたのは6ゲームのみ
   - 残り47ゲームは報酬配布がオフチェーン、他チェーン、またはトークン未発行

4. **BOMBトークンのCoinGecko登録がKaia未対応**
   - CoinGeckoにはBSC/TONのみ登録。Kaia上の`0x0ba2...`は未登録
   - 手動でアドレスを特定する必要があった

### ブロッカーではないが注意が必要なもの

- eth_getLogs の10,000件制限 → チャンク分割で対処可能
- CoinGecko APIレート制限 → キャッシュ + バッチ処理で対処可能
- 公開RPCの安定性 → 代替RPC `https://rpc.ankr.com/klaytn` も利用可能
